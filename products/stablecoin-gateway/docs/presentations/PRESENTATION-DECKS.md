## Slide 4: The Problem (Deep Dive)
**Title**: Why Credit Cards Suck for Merchants

**Problem 1: Fees Are Absurd**
- Interchange fee: 1.5-2.0% (goes to banks)
- Payment processor fee: 0.5-1.0% (Stripe, PayPal)
- Assessment fee: 0.1-0.15% (Visa, Mastercard)
- **Total: 2.9% + $0.30 per transaction**

Example:
- $100 sale ‚Üí merchant gets $96.80
- $1M/year sales ‚Üí $29,000 in fees üò±

**Problem 2: Settlement is Slow**
- Stripe: 2-7 days
- PayPal: 3-5 days
- ACH: 5-7 days
- **Cash flow killer** for small businesses

**Problem 3: Chargebacks Are Broken**
- Customer disputes charge ‚Üí merchant loses money + $15 fee
- Fraud rate: 0.5-1% of revenue
- Legitimate businesses lose $31B/year to chargebacks

**Problem 4: Geographic Limitations**
- 2B adults unbanked (can't use credit cards)
- Cross-border fees: 3-5% extra
- Currency conversion: another 2-3%

**Why Hasn't This Been Solved?**
- Credit card networks are a duopoly (Visa, Mastercard)
- High barriers to entry (compliance, banking relationships)
- Merchants have no choice (customers expect credit cards)

**Enter Stablecoins**: All the benefits of crypto, none of the volatility

## Slide 5: Our Solution (Technical Deep Dive)
**Title**: How We Actually Work (Under the Hood)

**Architecture**:

**Frontend** (Customer-Facing):
- Hosted checkout page (Vite + React)
- Wallet connection (wagmi + viem)
- Real-time status updates (Server-Sent Events)

**Backend** (API):
- Fastify (TypeScript)
- PostgreSQL + Prisma ORM
- JWT authentication
- Webhook delivery (with signatures)

**Blockchain Layer**:
- Ethereum + Polygon support
- USDC/USDT (ERC-20 tokens)
- ethers.js for blockchain interaction
- Alchemy/Infura for RPC access

**Security**:
- Non-custodial (merchant controls wallet)
- API keys + JWT tokens
- Rate limiting (5 requests/15min on auth)
- Webhook HMAC signatures
- Security headers (helmet)

**Flow**:
1. Merchant creates payment session via API
2. Customer redirects to hosted checkout
3. Customer connects wallet (MetaMask/WalletConnect)
4. Customer approves USDC transfer
5. Blockchain confirms transaction (30-120 seconds)
6. Webhook fires to merchant's server
7. Merchant fulfills order

**Key Technical Decisions**:
- **Why non-custodial?** No money transmitter license needed
- **Why stablecoins only?** No volatility risk
- **Why Polygon?** Low gas fees ($0.01 vs $5 on Ethereum)
- **Why TypeScript?** Type safety, great DX
